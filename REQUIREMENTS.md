# ふたりのノート - 要件定義書

## 1. プロジェクト概要

### コンセプト
遠距離恋愛中のカップル2人だけで使う、共有モバイルWebアプリ（PWA）。
**「ふたりだけのノート」**をデジタル化したような、温かみのある雰囲気が特徴。

### ターゲット
2人だけのプライベートな利用。一般公開や審査（App Store / Google Play）は行わない。
URLを共有してiPhoneのホーム画面に追加して使用する。

### 利用シーン
- 次に会う日の確認とカウントダウン
- 飛行機代の負担管理（どっちが出すか）
- お互いのスケジュール把握（出張、忙しい日など）
- 一緒に行きたい場所を貯める
- 一緒に観たい・聴きたいものを貯める

## 2. 機能一覧

### 2.1 ホーム画面
- 次に会う日までのカウントダウン表示
- 移動方向（誰→誰のところ）と飛行機代負担者の表示
- 行きたい場所・観たいものの件数サマリー
- 直近1週間のミニカレンダー
- 最近追加されたリストアイテムの表示
- ふたりのアイコン表示（ペア接続状態）

### 2.2 カレンダー画面
月表示のカレンダーに以下4種類の予定を色分け表示：

| 種類 | 用途 | 主な入力項目 |
|------|------|------|
| `visit` | 会いに行く（片方が移動） | 日付範囲、移動方向、飛行機代負担者、メモ |
| `trip` | ふたりで旅行（第三の場所で合流） | 日付範囲、行き先、飛行機代の扱い、メモ |
| `online` | オンラインデート | 日付、内容（観たい映画リストと紐付け可）、メモ |
| `anniversary` | 記念日・誕生日 | 日付、内容、繰り返し設定 |
| `personal` | 個人スケジュール（軽い共有） | 日付、所有者、ラベル（出張/忙しい等）、メモ |

**カレンダー機能の詳細：**
- 月間ビュー（横スワイプで月切り替え）
- 日付タップで詳細表示・編集
- 凡例表示（種類ごとの色）
- 今日の日付をハイライト
- 連続する会う期間は同じ色で帯のように連結表示

### 2.3 飛行機代管理
- カレンダーの`visit` / `trip`予定に紐付け
- 負担者の選択（わたし / かれし / それぞれ）
- 任意で金額を記録
- 累計集計表示（負担回数、合計金額）

### 2.4 行きたい場所リスト
- カード形式表示
- 項目：場所名、写真（任意）、カテゴリ（カフェ/レストラン/旅行先/デート/その他）、メモ、追加者
- ステータス：未訪問 / 訪問済み
- フィルタ：カテゴリ別、ステータス別、追加者別
- 旅行予定（`trip`）に紐付け可能

### 2.5 観たい・聴きたいリスト
- タブ切り替え：映画 / ドラマ / アニメ / 音楽 / 本
- 項目：タイトル、画像URL（任意）、追加者のコメント、追加者
- ステータス：未鑑賞 / 鑑賞済み
- オンラインデート予定（`online`）に紐付け可能

### 2.6 設定画面
- ふたりの名前とアイコン色設定
- 記念日（付き合った日）設定
- カップルコードの表示・再発行
- テーマカラー切り替え（任意）
- ログアウト

### 2.7 認証・ペアリング
- メールアドレス + パスワード（または Magic Link）でログイン
- 初回ログイン時に**カップルコード**を発行
- 相手がコードを入力するとペア成立
- データは「カップルID」に紐づけて保存
- 1カップル = 2ユーザーのみ

## 3. 画面構成・ナビゲーション

### ナビゲーション構造
下部固定タブ（4タブ）：
1. 🏠 **ホーム** - サマリー画面
2. 📅 **カレンダー** - 予定管理
3. 📋 **リスト** - 行きたい場所 / 観たいものを統合表示（上部タブで切替）
4. ⚙️ **設定** - アカウント・ペア管理

### 共通UI要素
- 右下フローティングアクションボタン（FAB）：コンテキストに応じた追加操作
- ボトムシート：予定・アイテム追加/編集はモーダルでなく下からせり上がるシート

## 4. データベース設計（Supabase）

### テーブル定義

#### `users`
| カラム | 型 | 説明 |
|--------|------|------|
| id | uuid | Supabase Auth と紐付け |
| email | text | メールアドレス |
| display_name | text | 表示名 |
| color | text | アイコンカラー（hex） |
| couple_id | uuid | 所属するカップルID（FK） |
| created_at | timestamptz | |

#### `couples`
| カラム | 型 | 説明 |
|--------|------|------|
| id | uuid | カップルID |
| invite_code | text | ペアリング用コード |
| anniversary_date | date | 記念日 |
| created_at | timestamptz | |

#### `events`（カレンダー予定）
| カラム | 型 | 説明 |
|--------|------|------|
| id | uuid | |
| couple_id | uuid | FK |

| end_date | date | 終了日（単日なら同じ） |
| traveler_id | uuid | `visit`時：移動する人のuser_id |
| host_id | uuid | `visit`時：受け入れる人のuser_id |
| destination | text | `trip`時：行き先 |
| flight_payer | text | `me` / `partner` / `split` / null |
| flight_amount | integer | 金額（任意、円） |
| linked_place_id | uuid | `trip`時：行きたい場所IDと紐付け |
| linked_media_id | uuid | `online`時：観たいものIDと紐付け |
| owner_id | uuid | `personal`時：誰の予定か |

| memo | text | メモ |
| created_by | uuid | 作成者 |
| created_at | timestamptz | |

#### `places`（行きたい場所）
| カラム | 型 | 説明 |

| category | text | カフェ/レストラン/旅行先/デート/その他 |
| photo_url | text | 写真URL（Supabase Storage）|
| memo | text | メモ |

| visited_at | date | 訪問日 |

#### `media`（観たい・聴きたいもの）
| カラム | 型 | 説明 |
|--------|------|------|
| id | uuid | |
| couple_id | uuid | FK |
| category | text | movie/drama/anime/music/book |
| title | text | タイトル |
| image_url | text | 画像URL（任意） |
| comment | text | 追加者のコメント |
| watched | boolean | 鑑賞済みフラグ |
| watched_at | date | 鑑賞日 |
| created_by | uuid | 追加者 |
| created_at | timestamptz | |

### RLS（Row Level Security）
- すべてのテーブルで`couple_id`によるアクセス制御を必須化
- 自分のカップルIDに紐づくデータのみ読み書き可能

## 5. 技術スタック

### フロントエンド
- **Next.js 14+**（App Router、TypeScript）
- **Tailwind CSS**（スタイリング）
- **Framer Motion**（アニメーション）
- **vaul**（ボトムシート）
- **react-swipeable**（スワイプ操作）
- **date-fns**（日付処理）
- **lucide-react**（アイコン）

### バックエンド・インフラ
- **Supabase**
  - PostgreSQL（DB）
  - Auth（認証）
  - Storage（画像保存）
  - Realtime（リアルタイム同期）
- **Vercel**（ホスティング・自動デプロイ）

### PWA化
- **next-pwa**（manifest.json + Service Worker自動生成）
- アイコン：180×180, 152×152, 192×192, 512×512 各サイズ用意
- スプラッシュスクリーン設定

### フォント
- **Klee One** または **Yomogi**（手書き風日本語）
- **Quicksand**（数字・日付用、丸み系サンセリフ）

## 6. デザインガイドライン

### 全体トーン
**かわいい・あたたかい・紙のノート風**

### カラーパレット
| 用途 | カラー | hex |
|------|------|------|
| 背景（メイン） | クリーム | `#FAF5EE` |
| 背景（カード） | ホワイト | `#FFFFFF` |
| ボーダー | ベージュ | `#EAE2D2` |
| メイン文字 | ブラウン | `#5C4538` |
| サブ文字 | ライトブラウン | `#A89070` |
| アクセント（会う日） | パステルピンク | `#FBE4E0` `#E8B5AB` |
| アクセント（行きたい場所） | セージグリーン | `#E4EDDF` `#6B8E5A` |
| アクセント（観たいもの） | ラベンダー | `#E5DFEC` `#7E6296` |
| わたしカラー | ピンク | `#FFEEF3` `#B0577A` |
| かれしカラー | パープル | `#E5DFEC` `#5B4374` |

### 形状・装飾
- 角丸：カード16-24px、ボタン12px、小要素8-10px
- 影：基本使わない。FABなど目立たせる箇所のみ淡い影
- ボーダー：0.5pxの細い線
- 投稿者タグ：カード上部に小さなピル形状で表示

### モバイル最適化
- 本文フォントサイズ：最低16px（iOSの自動ズーム防止）
- タップターゲット：最低44×44px
- セーフエリア対応：`env(safe-area-inset-*)`を使用
- ヘッダー固定スクロール
- 慣性スクロール有効

### アニメーション
- 画面遷移：左右スライド（ネイティブiOS風）
- タップフィードバック：0.1秒程度の縮小
- ボトムシート：下から上にスライドイン
- 重く感じないよう200-300ms程度に統一

## 9. 注意事項

### セキュリティ
- Supabase RLS必須。`couple_id`チェックを全テーブルで設定
- カップルコードは推測困難な形式（6-8文字英数字ランダム）
- 1つのカップルコードは1回使用で無効化

### プライバシー
- 2人だけが使う前提なので、招待制で他者の参加不可
- データは全てSupabaseに保存（ローカルストレージは認証トークンのみ）

### モバイル特有の留意点
- iPhoneのSafariでPWAとして動作させる
- `apple-mobile-web-app-capable` 等のメタタグ必須
- 16px未満のフォントは自動ズームを引き起こすので避ける
- `position: fixed`使用時はセーフエリア考慮
